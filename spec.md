# ato MVP — Specification (Phase 0)

## What we are building
**1日で動く、Web版PWA。完全に自分1人用。**

これは ato アプリの Phase 0、つまり「自分が30日使える最小機能版」である。
他人に見せる、課金する、認証する、これらはすべて Phase 1 以降。

## Why this scope
- 完璧を目指すと永遠に動かない
- 自分が触らないと、本当に必要な機能はわからない
- 動くものがあれば、文と質メンバーへの説明が10倍説得力を持つ

## Scope (作るもの)

### 1. 時間連動の背景
- 現在時刻に応じて、画面全体の背景色がグラデーションで変化
- 10時間帯のパレットを補間して滑らかに遷移
- 1分ごとに自動更新
- 画面下部に小さく時間帯名を表示(例: 「黄昏 — tasogare」)

### 2. ホーム画面(録音中心)
- 中央に大きな録音ボタン
- ボタンの上に問いかけ(時間帯ごとに変化:朝なら「今朝は、どんな空気でしたか」など)
- ボタンを触ると15秒録音開始 → 自動停止
- 録音中はリングが脈動する
- 録音終了後、Web Speech API で自動文字起こし
- 結果はLocalStorageに保存

### 3. 写真追加
- ホーム画面の隅に小さく「写真」ボタン
- タップするとカメラロール選択(input type="file" accept="image/*")
- 選んだ写真は LocalStorage に Base64 で保存
- 保存時の時刻と一緒に記録

### 4. 1日1問の問いかけ
- 22時前後にホーム画面に「今日の問い」が表示される
- 問いは事前に用意した50個からランダム選択(後でAI生成にする)
- 1行のテキスト入力欄
- スキップ可能(押し付けない)

### 5. 記録一覧
- ホーム画面下部にスクロールすると、過去の記録が時系列で並ぶ
- 各記録: 時刻、内容(声→テキスト/写真サムネ/1行回答)、その時の時間帯名
- タップすると詳細表示

## Out of Scope (作らないもの)
これらは絶対に Phase 0 では作らない:
- ❌ ユーザー登録、ログイン
- ❌ Supabase連携、サーバー保存
- ❌ Claude API による「気づき」生成(まずローカルで貯めるだけ)
- ❌ 自分マップ、星座可視化(データが貯まってから)
- ❌ 製本機能
- ❌ 通知(PWAの通知は Phase 1)
- ❌ 設定画面
- ❌ ダークモード切り替え(時間連動が機能を兼ねる)
- ❌ Suno API、音楽生成
- ❌ 家族に遺す機能

## Tech Stack
- **Next.js 15 App Router + TypeScript**
- **Tailwind CSS v4**
- **Web Speech API** — ブラウザ内蔵の音声認識(無料、APIキー不要)
- **MediaRecorder API** — 音声録音
- **LocalStorage** — データ保存(Phase 0 のみ)

## Data Model

```typescript
type Record = {
  id: string;              // UUID
  type: 'voice' | 'photo' | 'answer';
  createdAt: string;       // ISO 8601
  timeOfDay: string;       // '黄昏' など
  
  // type === 'voice'
  audioBlob?: string;      // Base64 (Phase 0)
  transcript?: string;     // 文字起こし結果
  
  // type === 'photo'
  imageBlob?: string;      // Base64
  
  // type === 'answer'
  question?: string;       // その日の問い
  answer?: string;         // ユーザーの1行回答
};
```

## Implementation Order (推奨)

Claude Code に Plan を出させる時、以下の順序を推奨する:

1. **プロジェクト初期化と Tailwind 設定**(色とフォントの定義)
2. **time-color.ts と TimeBackground コンポーネント**(世界観の核)
3. **storage.ts**(LocalStorage CRUD)
4. **ホーム画面の骨組み**(まだ機能なし、レイアウトだけ)
5. **RecordButton**(録音→文字起こし→保存まで)
6. **PhotoUpload**(写真選択→Base64→保存)
7. **QuestionCard**(1日1問の表示と回答保存)
8. **RecordList**(下部に記録一覧)
9. **記録詳細ページ**
10. **PWA対応**(manifest.json と icon)

各ステップ完了時に、必ず動作確認とコミット。

## Browser Compatibility

ターゲット: **Safari on iPhone (iOS 17+)**

理由: あなた自身が iPhone で使うため。後でホーム画面追加して使う前提。

注意点:
- Web Speech API は Safari では `webkitSpeechRecognition` を使う
- MediaRecorder の対応形式は Safari と Chrome で異なる(audio/mp4 vs audio/webm)
- LocalStorage の容量は 5MB 程度。写真の Base64 は重いので、画像は圧縮して保存

## Success Criteria

夜になる頃、以下が動いていれば MVP 成功:
- [ ] iPhone Safari で開ける
- [ ] 時間帯に応じて背景色が変わる
- [ ] 録音→文字起こし→保存ができる
- [ ] 写真を1枚追加できる
- [ ] 1日1問が出て、答えられる
- [ ] 過去の記録が下に並ぶ
- [ ] ブラウザ閉じても消えない

これだけで、ato の核心体験が動き始める。
